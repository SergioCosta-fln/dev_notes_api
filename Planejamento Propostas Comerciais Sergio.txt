PLANEJAMENTO

	PORPOSTAS: (Sistema de propostas comerciais)

	1. O que o porjeto precisa fazer?
		1.1. Produtos e Serviços
			- Listar os produtos/serviços
			- Pegar informações de um produto/serviço
			- Pegar informações de um determinado tipo de produto/serviço
			- Pegar informações de uma determinada forma de comercialização
			- Inserir um produto/serviço
			- Atualizar um produto/serviço
			- Deletar um produto/serviço
			
		1.2. Clientes
			- Listar os clientes
			- Pegar informações de um cliente
			- Pegar informações de um determinado tipo de pessoa
			- Inserir um cliente
			- Atualizar um cliente
			- Deletar um cliente
		
		1.3. Propostas
			- Listar as propostas
			- Pegar informações de uma proposta
			- Pegar informações de um cliente na proposta
			- Inserir uma proposta
			- Atualizar uma proposta
			- Deletar uma proposta
		
		1.4. Itens da Proposta (Produto e/ou Serviços)
			- Listar os itens da proposta
			- Pegar informações dos itens da proposta
			- Pegar informações de itens para um determinado produto/serviço
			- Inserir item na proposta
			- Atualizar um item da proposta
			- Deletar um item da proposta
			
			
	2. Qual a estrutura de dados?
		2.1. Produtos e Serviços
			- Local para armazenar os produtos/serviços
				-- id
				-- descricao
				-- descritivo
				-- valorVenda			(valor de venda)
				-- tipos				(Tipos de serviços -> SaaS - Locação - Serviço Técnico - Produto)
				-- formaComercializacao	(Venda Direta - Mensalidade - Trimestralidade, Semestralidade - Anualidade)
		
		2.2. Clientes
			- Local para armazenar os clientes
				-- id
				-- nome
				-- tipoPessoa	(Física - Jurídica)
				-- cpf_cnpj
				-- cep
				-- endereco
				-- numero
				-- complemento
				-- bairro
				-- cidade
				-- estado
				-- pais

		2.3. Propostas
			- Local para armazenar as propostas
				-- id
				-- codigo
				-- assunto
				-- data
				-- dataValidade
				-- idCliente
				
		2.4 - Itens da Proposta
			- Local para armazenar os itens da proposta
				-- id
				-- idProposta
				-- idProduto
				-- quantidade
				-- valorUnitario
				-- desconto
				-- valorTotal	((quantidade * valorUnitario) - desconto)
				

	3. Quais os endpoints?

			-- (METODO) /url (PARÂMETROS)
			-- (GET) /api/product

			-- (PUT) /api/product/123 		(parametros -> nome produto, descritivo, tipos de serviços, forma de comercialização)
			-- (DELETE) /api/product/123	(id)
